汇编语言

课程重点：如何利用硬件系统的编程结构和指令集有效灵活的控制系统进行工作

存储器：

​ 存取器被划分为若干个存储单元，每个存储单元从0开始顺序编号

  • RAM
  • ROM

BIOS： 基本输入输出系统

内存地址空间： 与CPU总线相连，CPU通过控制线对它们发出读写命令的多个物理上独立的存储芯片组合成的一个逻辑存储器

物理地址： 所有内存单元构成的存储空间是一个一维的线性空间，每一个内存单元都有一个唯一的物理地址

​ 由于8086CPU一次能计算的位数小于地址总线一次能传输的位数，不能表达所有的物理地址，所以需要地址加法器。

  1. CPU中的相关部件提供两个16位地址，一个称为段地址，一个称为偏移地址，
  2. 段地址和偏移地址通过内部总线送入一个称为地址加法器的部件
  3. 地址加法器将两个16位地址合成为一个20位的物理地址
  4. 地址加法器通过内部总线将20位物理地址送入输入输入控制电路
  5. 输入输出控制电路将20位物理地址送上地址总线
  6. 20位物理地址被地址总线传送到存储器

​ 地址加法器采用物理地址=基础地址（段地址x16(1H)）+偏移地址的方法（一个16进制数占4位，段地址4个16进制数x1H=5个16进制数=20位）

寄存器： CPU里存储数据的器件 8086CPU所有寄存器是16位

  • 通用寄存器
    • AX可以分为AH（高位）和AL（低位）
  • 段寄存器：提供段地址
    • CS（Code Segment）,DS（Data）,SS（Stack）,ES（Extra）
  • 指令指针寄存器
    • IP

高低位： 数据从高位到低位（10000，1是高位）

指令和数据：指令和数据是应用上的概念，在内存或硬盘上没有任务区别，都是二进制信息，根据不同总线区分

CPU：

​ CPU由运算器，控制器，寄存器组成，靠内部总线相连

​ CPU的工作是取指执行，需要指令和数据，存放在内存，CPU无法执行读取存储在硬盘中的数据，需要先读取到内存。

​ CPU要进行数据的读写，必须和外部器件（芯片）进行三类信息的交互：

  • 存储单元的地址（地址信息）
  • 器件的选择，读或写命令（控制信息）
  • 读或写的数据（数据信息）

​ 通过外部总线传输信息，逻辑上划分为：

  • 地址总线

    一个CPU有N根地址总线，则这个CPU的地址总线宽度为N，可以寻址2的N次方个内存单元（Byte）

  • 数据总线

    数据总线的宽度决定CPU和外界的数据传送速度

  • 控制总线

    控制总线是一些不同控制线的集合

​ 读取数据的过程：CPU向内存发送数据地址，发送读取控制，内存发送数据，CPU获得数据

​ 写入数据的过程：CPU向内存发送数据地址，发送写入控制，发送写入数据，内存覆盖原有数据

​ 每一种CPU都有自己的汇编指令集

​ 8086CPU工作过程：

  • 从CS:IP指向内存单元读取指令，读取的指令进入指令缓冲器

  • IP = IP +所读取指令的长度，从而指向下一条指令

  • 执行指令

    8086CPU加电启动或复位后（刚开始工作），CS=FFFFH,IP=0000H，即开机启动后，CPU从内存FFFF0H单元中读取指令执行，该指令为开机后执行的第一条指令。

DEBUG:

  • R 查看，改变CPU寄存器的内容
  • D 查看内存中的内容
  • E 改写内存中的内容
  • U 将内存中的机器指令翻译成汇编指令
  • T 执行一条机器指令
  • A 以汇编指令的格式在内存中写入一条机器指令

机器语言：

​ 机器语言是机器指令的集合 由0和1组成，对人类不友好，出错难以发现

汇编语言：

​ 底层语言，直接在硬件之上工作的编程语言，效率高，学习汇编语言有助于了解计算机底层运作方式，计算机只能读懂机器指令，所以需要将汇编指令编译成机器指令 与机器语言对比，汇编语言与人类语言接近，便于阅读和记忆

​ 组成：

  • 汇编指令（机器码的助记符）
    • mov （8086CPU mov不能设置CS,IP的值）
    • add
    • sub
    • jmp
      • 同时修改CS:IP的值 （jmp 2AE3:3）
      • 仅修改IP （jmp ax 类似于 mov IP ，ax）
  • 伪指令（由编译器执行）
  • 其他符号（由编译器识别）

​ 汇编语言的核心是汇编指令，它决定了汇编语言的特性
