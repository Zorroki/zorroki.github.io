Tomcat与Servlet3.0+

Tomcat

Catalina 是tomcat的servlet容器，实现了servlet和jsp规范。

Coyote 是tomcat的web server，负责监听请求和响应。

CATALINA_HOME: tomcat所在目录

CATALINA_BASE: 当前使用配置所在目录

在idea中运行tomcat，idea并不会直接把编译后的项目部署到tomcat的webapps文件夹下，而是：

  1. 在${user.home}/.IntelliJIdea/system/tomcat下为每一个被部署的artifact创建一个名为Unnamed_<项目名称>的文件夹
  2. 复制tomcat目录下的配置文件到该目录下
  3. 设置配置文件里的docBase参数指向编译后的输出文件夹或war （配置文件路径：conf/Catalina/localhost）
  4. 启动tomcat，设置参数CATALINA_BASE=${user.home}/.IntelliJIdea/system/tomcat/Unnamed_<项目名称>

Servlet 3.0+

Spring WebApplicationInitializer提供了在Servlet 3.0+容器里编程式的配置Spring DispatcherServlet 和 ContextLoaderListener的方式，实现了WebApplicationInitializer接口的类会在Servlet容器启动阶段被调用

原理是：

​ Servlet 3.0新增了一个接口ServletContainerInitializer，接口实现类会在容器启动阶段被调用和传入一个ServletContext，从而实现编程式的注册(Servlet，Listener)，根据Servlet规范,必须在jar包里的META-INF/services/javax.servlet.ServletContainerInitializer文件里声明实现类。Spring 的SpringServletContainerInitializer实现了该接口,并在spring-web*.jar下声明。

​ SpringServletContainerInitializer类带有注解@HandlerTypes(WebApplicationInitializer.class),这意味着Servlet容器会扫描寻找实现了WebApplicationInitializer接口的实现类，在调用onStartup方法时和ServletContext一起传入。
